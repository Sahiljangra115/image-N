# RAG Resources

## Knowledge

- [Paper: "Retrieval-Augmented Generation for Knowledge-Intensive NLP Tasks" — Lewis et al., 2020 (arXiv 2005.11401)](https://arxiv.org/abs/2005.11401)
  The original paper that coined RAG. The canonical primary source. Use for: the precise definition, the parametric vs non-parametric memory framing, and being able to say "I read the source," not a blog about it. Dense; read the abstract and intro first, return to the rest later.
- [Guide: "Retrieval-Augmented Generation (RAG)" — Pinecone](https://www.pinecone.io/learn/retrieval-augmented-generation/)
  High-trust, beginner-friendly walkthrough of the pipeline with diagrams. Use for: a clear mental model of retrieve-then-generate and how vector search fits in.
- [Guide: "What is RAG?" — AWS](https://aws.amazon.com/what-is/retrieval-augmented-generation/)
  Vendor-neutral-ish plain-English explainer. Use for: a second framing of the same ideas (interleaving sources aids retention).
- [Reference: "Retrieval-augmented generation" — Wikipedia](https://en.wikipedia.org/wiki/Retrieval-augmented_generation)
  Use for: quick term lookups and an overview of RAG variants (Self-RAG, Corrective RAG, GraphRAG) once the basics are solid.

## Wisdom (Communities)

- [r/LocalLLaMA](https://www.reddit.com/r/LocalLLaMA/)
  High-signal subreddit where people actually build RAG systems and share what breaks in practice. Use for: chunking/embedding/retrieval troubleshooting and seeing real architecture debates.
- [r/MachineLearning](https://www.reddit.com/r/MachineLearning/)
  Use for: grounding in the wider ML field (helps with the non-LLM flagship later) and paper discussion.
